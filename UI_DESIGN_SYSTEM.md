# 极速箱 (Jisuxiang) - 现代极简主义 UI 设计系统指南

本文档记录了极速箱项目从早期的“紫黑渐变与发光阴影”风格向**现代极简主义 (Modern Minimalist)** 风格的全面重构过程与设计规范。本规范旨在为后续的组件开发和页面维护提供一致的审美指导。

## 1. 设计理念 (Design Philosophy)

本次 UI 改造的核心理念是**克制、专业与内容优先**，设计灵感来源于 Vercel、Shadcn UI 等业界前沿的开发者工具平台。

*   **摒弃冗余装饰**：去除了所有不必要的渐变色 (`bg-gradient`)、大面积投影 (`shadow-lg`) 和悬浮位移 (`translateY`)。
*   **边框驱动 (Border-driven)**：依赖细腻的 1px 边框来划分信息层级和区域，而非依靠沉重的背景色差或阴影。
*   **高对比度**：利用纯黑与纯白的极致对比来引导用户视觉焦点，提供极佳的阅读体验。

## 2. 色彩系统 (Color System)

项目的色彩体系已在 `src/app/globals.css` 中进行了彻底重写，统一使用 CSS 变量。通过拉开背景与卡片的明暗对比，减少对边框的硬依赖，提升整体“呼吸感”。

### 深色模式 (Dark Mode - 默认)
深色模式不再使用偏蓝/紫的暗色，而是采用了极致的纯黑与深灰对比。
*   **背景色 (`--color-bg-main`)**: `#000000` (纯黑背景，提供深邃的沉浸感)
*   **卡片底色 (`--color-bg-card`)**: `#09090B` (微灰卡片，自然浮现于纯黑背景之上)
*   **主色调 (`--color-primary`)**: `#FAFAFA` (纯白，用于主按钮和核心强调元素)
*   **边框色 (`--color-border`)**: `rgba(255, 255, 255, 0.1)` (极为克制的极细边框)

### 浅色模式 (Light Mode)
浅色模式强调干净、通透与层级感。
*   **背景色 (`--color-bg-main`)**: `#FAFAFA` (极浅灰背景)
*   **卡片底色 (`--color-bg-card`)**: `#FFFFFF` (纯白卡片，自然浮现)
*   **主色调 (`--color-primary`)**: `#18181B` (深沉的近黑色，避免绝对纯黑带来的视觉疲劳)
*   **边框色 (`--color-border`)**: `rgba(9, 9, 11, 0.1)`

## 3. 核心组件规范 (Component Guidelines)

开发新工具或新页面时，请严格遵守以下组件类名和样式规范，**严禁在代码中硬编码渐变色或非标准阴影**。

### 卡片 (Cards - `.card`)
*   **外观**：圆角 `rounded-xl`，背景色随主题，带有微弱阴影 `shadow-sm` 和 `1px` 边框。
*   **交互**：悬停时仅轻微改变背景色 (`bg-secondary`) 和加深边框透明度，**严禁使用上浮 (`-translate-y`) 动画**。

### 按钮 (Buttons)
*   **主按钮 (`.btn-primary`)**：
    *   使用纯色背景（深色模式下为白色，浅色模式下为黑色）。
    *   包含少量内部阴影 `shadow-sm` 增加质感。
    *   悬停时背景色轻微变暗/变亮，**不再使用放大 (`scale`) 和渐变**。
*   **次要按钮 (`.btn-secondary`)**：
    *   透明背景，通过细边框界定轮廓。
    *   悬停时背景变为次级背景色。

### 输入框 (Inputs - `.search-input`)
*   **外观**：透明背景，依靠细边框界定。
*   **交互**：获取焦点 (`focus`) 时，边框颜色变为当前文字的主色，提供清晰但不刺眼的反馈。

## 4. 历史遗留样式的清洗细节

在本次改造中，我们对项目历史遗留的、破坏极简风格的 Tailwind 类名进行了全量清洗：

1.  **废弃的类名**：
    *   `bg-gradient-to-r` (全部移除，替换为纯色 `bg-[rgb(var(--color-primary))]`)
    *   `from-[#6366F1] to-[#8B5CF6]` / `from-indigo-500 to-purple-500` (全部移除)
    *   `shadow-[#6366F1]/20` / `shadow-lg` (发光特效已替换为克制的 `shadow-sm border-[var(--color-border)]`)
2.  **受影响的范围**：
    *   全局 CSS: `globals.css`
    *   首页: `src/app/page.tsx`
    *   公共组件: `src/components/BackToTop.tsx` 等。
    *   子工具页面: 包括不仅限于二维码生成器、代码格式化、图片水印、正则测试等十余个强依赖按钮高亮的页面。

## 5. 后续开发建议

*   **保持克制**：在添加新功能时，先考虑是否能用现有的边框和字重来区分信息，而不是盲目增加颜色。
*   **使用语义化变量**：始终使用 `rgb(var(--color-*))` 的形式引用颜色，确保在深浅模式下都能完美自适应。
*   **留白即设计**：现代极简风格高度依赖合理的内外边距 (`padding` / `margin`)。遇到拥挤的界面，优先增加间距，而非添加分割线。面，优先增加间距，而非添加分割线。
